# Автозаполнение полей урока из плана

Дата: 2026-06-02
Ветка: feature/plan-result-redesign (или новая feature-ветка)

## Проблема

При загрузке урока (аудио/видео) пользователь вручную вписывает **тему урока**
и **ключевые понятия плана** (по строке). Это ручная работа, которую можно убрать:
у пользователя обычно уже есть план урока (docx/pdf/xlsx/txt или просто текст).

## Цель

Дать возможность приложить план урока (файлом **или** вставленным текстом),
автоматически извлечь из него **тему** и **ключевые понятия** и подставить их
в поля формы. Пользователь проверяет/правит значения и запускает анализ как
обычно. Остальной пайплайн анализа (STT, диаризация, LLM-анализ, covered/missing,
структура, вопросы, баланс речи) **не меняется**.

## Решения по объёму (что НЕ делаем)

- **Не** меняем `POST /lessons/upload` — он по-прежнему принимает готовые
  `title` и `key_concepts`.
- **Не** добавляем анализ соответствия плану (сравнение этапов/целей) — выбран
  минимальный вариант «автозаполнение полей».
- **Не** трогаем модели БД и не делаем миграций — извлечённые поля живут только
  в форме, затем уходят в обычный upload.
- Ручной ввод темы/понятий **остаётся** как запасной путь.

## Архитектура

### Бэкенд

**Новый модуль `backend/app/plan_extract.py`:**

```python
def extract_plan_fields(plan_text: str, language: str) -> tuple[LLMPlanExtract, str]:
    """Зовёт Claude и возвращает (распарсенные поля, сырой ответ)."""
```

- Системный промпт (cache_control: ephemeral) просит вернуть СТРОГО JSON
  `{"title": str, "key_concepts": [str]}` — без markdown-обёрток.
- Промпт инструктирует: тема — короткая формулировка темы урока; key_concepts —
  ключевые понятия/термины, которые по плану должны прозвучать (только из плана,
  без воды), на языке плана.
- Парсинг через существующий `llm_utils.extract_json`, валидация
  `LLMPlanExtract.model_validate`.
- Модель из `ANTHROPIC_MODEL`. Если ключ не задан — понятная ошибка.

**Новый эндпоинт `POST /lessons/plan-extract` (синхронный, как `/plans/check`):**

```python
@app.post("/lessons/plan-extract", response_model=PlanExtractOut)
async def extract_plan_endpoint(
    language: str = Form("ru"),
    text: str = Form(""),
    file: UploadFile | None = File(None),
    db: Session = Depends(get_db),
):
```

- `language` ∈ {kk, ru}, иначе 400.
- Если приложен `file` — он в приоритете над `text`:
  - проверка расширения по `ALLOWED_PLAN_EXT` (иначе 400),
  - чтение с cap `MAX_PLAN_UPLOAD_MB` (иначе 413),
  - `doc_extract.extract_text(filename, raw_bytes)`.
- Иначе берём `text.strip()`.
- Если пусто и там, и там → 400 «Нужен текст плана или файл».
- Если извлечённый текст пуст → 400 «Не удалось извлечь текст из плана».
- Зовём `extract_plan_fields`, возвращаем `PlanExtractOut{title, key_concepts}`.
- Запись в БД не создаётся (это вспомогательный разбор, не история).

### Схемы (`schemas.py`)

```python
class LLMPlanExtract(BaseModel):
    """Схема ответа Claude при извлечении полей из плана урока."""
    title: str = ""
    key_concepts: List[str] = []

class PlanExtractOut(BaseModel):
    """Ответ POST /lessons/plan-extract — подставляется в форму."""
    title: str = ""
    key_concepts: List[str] = []
```

Изменений в `models.py` нет, миграция не требуется.

### Фронтенд

**`api.js`** — новая функция `extractPlanFields({ file, text, language })`:
шлёт `multipart/form-data` (`file` и/или `text`, `language`) на
`POST /lessons/plan-extract`, возвращает `{ title, key_concepts }`.

**`UploadForm.jsx`** — блок «План урока (необязательно)» над полями темы/понятий:
- отдельный `<input type="file">` (docx/pdf/xlsx/txt) — НЕ путать с аудио/видео;
- textarea «…или вставьте текст плана»;
- кнопка **«Разобрать план → заполнить поля»** — запускает разбор явно
  (и для файла, и для текста — единое предсказуемое поведение);
- во время запроса: статус «Разбираю план…», кнопки заблокированы.

Поведение по ответу:
- **перезаписываем** локальный стейт `title` (если непустой) и `keyConcepts`
  (понятия джойним через `\n`);
- ошибку разбора показываем в `form-error`, форму не ломаем — поля можно
  заполнить руками;
- повторный разбор перезаписывает поля заново.

Сабмит формы — без изменений: уходит в `/lessons/upload` с текущими
(возможно отредактированными) значениями.

## Крайние случаи

| Случай | Поведение |
|--------|-----------|
| Файл плана > `MAX_PLAN_UPLOAD_MB` | 413, сообщение |
| Неподдерживаемый формат файла | 400, сообщение |
| Пусто: ни файла, ни текста | 400 «Нужен текст плана или файл» |
| Извлечённый текст пуст | 400 «Не удалось извлечь текст из плана» |
| `ANTHROPIC_API_KEY` не задан / ошибка модели | понятная ошибка, поля без изменений |
| Выбраны и файл, и текст | приоритет у файла |
| Повторный разбор | перезаписывает поля |
