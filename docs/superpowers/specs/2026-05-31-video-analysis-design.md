# Дизайн: анализ урока по видеозаписи

**Дата:** 2026-05-31
**Статус:** утверждён, готов к плану реализации
**Автор:** обсуждение пользователь + Claude

## Цель

Дать возможность загружать **видеозаписи** урока (а не только аудио), извлекать
из них звуковую дорожку и прогонять по существующему пайплайну анализа. Метрики,
дашборд, схема БД и логика анализа ролей не меняются — расширяется только «вход».

## Решения (зафиксированы при брейншторминге)

| Вопрос | Решение |
|---|---|
| Что используем из видео | **Только звук.** Видеоряд не анализируем. |
| Форматы видео | `mp4`, `mov`, `mkv`, `webm` |
| Лимит размера файла | Мягкий, **~2 ГБ**, конфигурируемый через `MAX_UPLOAD_MB` (env, дефолт `2048`) |
| Подход к извлечению | **Нормализация любого входа в WAV** (Подход A) |
| Оригиналы файлов | Оставляем на диске (как сейчас с аудио); временный WAV удаляется после обработки |

## Подход A: нормализация входа в WAV

Любой загруженный файл (аудио **или** видео) приводится одним шагом к стандартному
`16 кГц, моно, WAV` во временный файл, после чего пайплайн идёт без изменений.
Whisper и pyannote читают этот WAV.

Почему A (а не «извлекать только для видео» или «whisper читает видео сам»):
- единый путь кода — меньше развилок, проще тестировать;
- `16 кГц моно` — ровно то, что нужно и Whisper, и pyannote; заодно нормализуем
  «капризное» аудио;
- whisper и pyannote получают **один и тот же** источник звука → нет риска
  рассинхрона таймкодов между STT и диаризацией.

Издержки: лишний (быстрый) шаг ffmpeg и для обычного аудио; временный WAV ~115 МБ
на час записи (удаляется после обработки).

## Поток данных

```
upload (audio | video)
  → валидация: расширение ∈ {аудио ∪ видео}, размер ≤ лимита (стрим с обрывом)
  → Lesson(status=pending)
  → фон process_lesson:
        wav = ensure_wav(original)        # ffmpeg → 16кГц моно WAV (temp)
        stt = transcribe_and_diarize(wav) # whisper + pyannote — без изменений
        analysis = analyze(stt)           # Claude — без изменений
        save AnalysisResult, status=done
        finally: удалить temp WAV
```

## Компоненты и изменения

### Новый модуль `backend/app/media.py`
Единственная задача — привести медиафайл к WAV.

- `ensure_wav(src_path: str) -> str`
  - запускает `ffmpeg -y -i <src> -vn -ac 1 -ar 16000 -f wav <out>`;
  - `<out>` — временный файл (`tempfile`), путь возвращается вызывающему;
  - при ненулевом коде возврата ffmpeg или пустом результате кидает исключение с
    понятным сообщением (например, `RuntimeError("Не удалось извлечь аудио из видео")`).
  - `-vn` отбрасывает видеопоток; для аудиофайлов шаг просто перекодирует в WAV.

### `backend/app/main.py`
- Множества расширений:
  - `AUDIO_EXT = {.mp3, .wav, .m4a, .ogg, .flac}` (как сейчас),
  - `VIDEO_EXT = {.mp4, .mov, .mkv, .webm}` (новое),
  - `ALLOWED_EXT = AUDIO_EXT | VIDEO_EXT`.
- Лимит размера: `MAX_UPLOAD_MB = int(os.getenv("MAX_UPLOAD_MB", "2048"))`.
  - При сохранении файла копируем потоково с подсчётом байт; при превышении лимита
    прерываем запись, удаляем частичный файл и отвечаем `413` с понятным текстом.
  - (Опционально, как быстрый отказ — проверка заголовка `Content-Length`, но
    основной контроль — по факту записанных байт, т.к. заголовок может врать.)
- `process_lesson`:
  - `wav_path = ensure_wav(audio_path)` перед транскрипцией;
  - передаём `wav_path` в `transcribe_and_diarize`;
  - в `finally` удаляем `wav_path` (temp), оригинал не трогаем.

### `backend/app/transcription.py`
Без изменений — получает готовый WAV-путь.

### `frontend/src/components/UploadForm.jsx`
- В `<input type="file">` расширить `accept` видео-MIME/расширениями.
- Обновить текст-подсказку (какие форматы поддерживаются).
- Опционально: клиентская проверка размера с дружелюбным предупреждением до отправки.

### Конфигурация и документация

- `docker-compose.yml`: пробросить `MAX_UPLOAD_MB` в окружение backend.

## Обработка ошибок

| Случай | Поведение |
|---|---|
| Неподдерживаемое расширение | `400` (существующий паттерн) |
| Превышен лимит размера | `413`, понятный текст, частичный файл удаляется |
| ffmpeg не извлёк звук (битый файл / нет аудиодорожки) | урок `status=error`, сообщение «Не удалось извлечь аудио из видео» (ловится существующим `try/except` в `process_lesson`) |
| Пустой транскрипт после распознавания | как сейчас — `status=error` |

## Тестирование

В проекте сейчас нет тестов — заводим минимальный `pytest` для backend.

- **`ensure_wav` (интеграционный, ffmpeg есть в контейнере):**
  - генерируем ffmpeg-ом крошечный клип (≈1 сек: тон + чёрный кадр), проверяем,
    что на выходе валидный непустой WAV `16 кГц/моно`;
  - клип **без аудиодорожки** → ожидаем внятную ошибку.
- **Лимит размера:** юнит на стрим-хелпер копирования — на входе больше лимита
  обрывает запись и сигнализирует превышение.
- **Валидация расширений:** видео-расширения принимаются, мусорные — отклоняются.
